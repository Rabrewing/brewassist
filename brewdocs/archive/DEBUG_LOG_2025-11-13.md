# Debug Log - 2025-11-13

This log tracks the issues we've encountered and the steps taken to resolve them while debugging the BrewAssist DevOps AI Cockpit.

---

### **Issue 1: Buttons and Chat Not Working (Timestamp: ~05:30 UTC)**

- **Problem:** The "Send" button, side buttons, and "Enter" key in `chat.html` were unresponsive.
- **Investigation:**
  - Reviewed `chat.html`, `chat.js`, and `server.js`.
  - Discovered that `chat.html` was not loading `chat.js`.
- **Fix:** Added `<script src="chat.js" defer></script>` to `chat.html`.
- **Status:** Resolved the initial button issue but uncovered underlying API failures.

---

### **Issue 2: API Failures for AI, Supabase, and Guide (Timestamp: ~05:45 UTC)**

- **Problem:** The AI chat, Supabase Push, and BrewGuide features were not working. Browser logs showed `404 Not Found` errors for `/api/llm`, `/api/supa`, and `/api/guide`.
- **Investigation:**
  - Created a `pages/api/test.ts` route, which worked correctly. This proved the Next.js server was running but failing to compile the specific, more complex API files.
  - Identified a potential syntax error (`new Promise < string >`) in `llm.ts` that could cause a silent compilation failure.
- **Fix:** Corrected the syntax in `llm.ts` to `new Promise<string>`.
- **Status:** This fix led to a new, more severe issue.

---

### **Issue 3: Server Hanging (Timestamp: ~06:30 UTC - Current)**

- **Problem:** After fixing `llm.ts`, the `server.js` process began hanging. `localhost:5000/chat-ui/chat.html` would no longer load. `curl` requests to the server would hang indefinitely.
- **Investigation:**
  - Confirmed `server.js` starts without errors.
  - Added a `/test-server` route to `server.js`, which worked. This isolated the problem to the middleware.
  - Isolated the conflict to the `http-proxy-middleware`. The server worked when the proxy was commented out.
  - Re-enabled the proxy with logging, but the server began hanging again, and no logs were printed, not even from a new top-level logger.
  - **Further Investigation (Timestamp: ~06:50 UTC):** Discovered that `app.use(cors());` was the direct cause of the server hanging. When commented out, `express.static` middleware functioned correctly.
  - **Fix:** Temporarily commented out `app.use(cors());` in `server.js`.
  - **Status:** Server no longer hangs, `chat.html` loads.

---

### **Issue 4: API Calls Still Failing (Timestamp: ~07:00 UTC)**

- **Problem:** After fixing the server hang, API calls (`/api/llm`, `/api/supa`, `/api/guide`) were still not working.
- **Investigation:**
  - Added a top-level logger to `server.js` and reordered middleware to ensure proxy was hit first.
  - Logs showed requests reaching `server.js` but _not_ being processed by `http-proxy-middleware`.
  - Confirmed `http-proxy-middleware` dependency was present in `package.json`.
  - Simplified `http-proxy-middleware` configuration to its bare minimum.
  - **Breakthrough:** Proxy started working, forwarding requests to `localhost:3000`. Next.js server (`npm run dev`) then reported `POST /llm 404`. This confirmed the problem was now isolated to the Next.js API routes.
  - **Fix:** Proxy middleware in `server.js` simplified and reordered.

---

### **Issue 5: Incorrect API Endpoint in Frontend (Timestamp: ~07:30 UTC)**

- **Problem:** `chat.js` was calling `/api/llm` for AI prompts, but the user's diagnostic guide and `brewassist.ts` indicated `/api/brewassist` was the correct main entry point.
- **Investigation:**
  - Reviewed `brewassist.ts` and confirmed it's the intended AI entry point.
  - Noted `opencode-grok` CLI was not found during system checks.
  - User confirmed `.env.local` flags were set.
  - Ran `brewstatus.sh` (successful) and checked for `brewtest.sh` (not found).
  - User updated `.env.local` with new variables.
- **Fix:** Changed `fetch("/api/llm", ...)` to `fetch("/api/brewassist", ...)` in `chat.js`.

---
