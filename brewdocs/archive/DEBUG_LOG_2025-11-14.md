### **Case Study: The Negative Modulo Bug**

- **Symptom:** The `/api/brewassist` endpoint was crashing with a `TypeError` because the `tier` variable was `undefined`.
- **Investigation:**
  - Through detailed logging, we traced the issue to the `randomEmotion` function in `lib/utils/emotion.ts`.
  - The logs revealed that the hashing function produced a negative number (`hash=-1327670739`).
  - The code used the standard modulo operator (`%`) to get an index from the `ORDER` array: `h % ORDER.length`.
  - **Root Cause:** In JavaScript, the modulo operator with a negative number produces a negative result (e.g., `-1327670739 % 7 = -3`). This resulted in an attempt to access `ORDER[-3]`, which is `undefined`.
- **Resolution:** The fix was to ensure the index is always non-negative. The formula was changed to `((h % ORDER.length) + ORDER.length) % ORDER.length`.
- **Next Steps:** Apply the fix to `lib/utils/emotion.ts` and re-test the `/assist` command.

### **Final Fix: Frontend Command Parsing**

- **Symptom:** After fixing the "Negative Modulo Bug", `/api/brewassist` still returned a 500 error, but without a stack trace.
- **Investigation:**
  - Reviewed `public/chat.js` and discovered the root of the command-handling problem.
  - The frontend was not parsing slash commands. It was sending the _entire_ input string (e.g., `/assist --mode auto "..."`) as a single `prompt` to `/api/brewassist`.
  - This caused the backend `brewassist.sh` script to receive incorrect arguments, leading to an execution error, which was caught by the `try...catch` block in `pages/api/brewassist.ts`, resulting in a generic 500 error.
- **Resolution:** The event listener in `public/chat.js` was rewritten to be command-aware. It now checks if the input starts with `/`. If so, it parses the command and arguments and sends them to the central `/api/router`. If not, it sends the prompt to `/api/brewassist`.
- **Conclusion:** This unifies the command structure and ensures the backend scripts receive the correct, parsed arguments.
- **Next Steps:** Final confirmation test of all commands.
