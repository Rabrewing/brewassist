# 1. 🎯 Purpose of S4.4

S4.4 introduces the **first real layer of BrewAssist synthetic personality** — enabling BrewAssist to communicate like a natural collaborator instead of a stateless API.

**Key goals:**
- Create a stable persona definition for RB Mode  
- Make BrewAssist conversational, memory-aware, and emotionally adaptive  
- Add safety gates (soft-stop) for risky requests  
- Introduce a 12-turn context window  
- Log personality actions via BrewLast  
- Prepare for S4.5 Self-Maintenance Sandbox  

This is what transforms BrewAssist from a “tool executor” into a **dev partner.**

# 2. 🧠 Architecture Overview

## **2.1 Personality Registry**  
File: `lib/brewPersona.ts`  
Defines stable JSON personality profiles:

- **rb** → your mode  
- **default** → fallback  
- **research** → outbound researcher mode  
- **strict** → enterprise compliance mode  

Each profile includes:

| Field | Purpose |
|-------|---------|
| tone | conversational / neutral / enterprise |
| emotionTier | 1–5 emotional expressiveness |
| energy | neutral / high / low |
| safetyMode | soft-stop / hard-stop / none |
| memory | context window size |
| rules | do/don’t instructions for the engine |

---

## **2.2 Personality Engine**  
File: `lib/brewAssistPersonality.ts`

Performs:

1. **Tone adaptation**  
2. **Emotion tier selection**  
3. **Safety route evaluation**  
4. **Context stitching (last 12 turns)**  
5. **Persona-aware summarization**  

---

## **2.3 Safety Layer**  
File: `lib/brewAssistSafety.ts`

Handles:

- Soft-stop  
- Hard-stop  
- Override flow  
- Risk classification  
- Structured warning responses  

---

## **2.4 Context Window**  
File: `lib/brewContextWindow.ts`  

Stores 12-turn rolling memory (S4.4 spec).

No SQL DB used — BrewLast remains FS-only.

---

## **2.5 API Endpoints Added**

| Route | Purpose |
|-------|---------|
| `/api/brewassist-persona` | Main personality interface |
| `/api/brewassist-health` | Returns personality + safety status |
| `/api/brewassist-suggest` | AI improvement suggestions for S4.5 |
| `/api/edit-file` | Optional persona-driven editing flow |
