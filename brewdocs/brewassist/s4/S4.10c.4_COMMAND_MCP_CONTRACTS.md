# S4.10c.4: Command + MCP Contract Tables

This document contains the authoritative contract tables for BrewAssist commands and MCP (Toolbelt) tools. These tables are the single source of truth for policy enforcement.

## 1) Command Contract Table

Each `/command` must have: intent, scope, personas, tier, tool usage, brewtruth expectations, failure contract.

| Command     | Intent Category | Scope Allowed (Admin/Customer) | Personas Allowed         | Required Tier | Toolbelt Use | BrewTruth Required | Output Contract                                       | Failure Contract                               |
|-------------|-----------------|--------------------------------|--------------------------|--------------:|--------------|--------------------|-------------------------------------------------------|------------------------------------------------|
| `/task`     | SUPPORT         | Admin ✅ / Customer ✅         | Admin, Support, Customer |             1 | No           | Yes (light)        | `{ ok, route:"brewassist", content, truth?, policy? }` | `TOOLBELT_*` / `SCOPE_BLOCKED` / `BAD_INPUT`     |
| `/doc`      | DOCS_KB         | Admin ✅ / Customer ✅         | Admin, Support, Customer |             1 | No           | Yes (light)        | returns docs summary + links (internal)               | `DOC_NOT_FOUND` / `SCOPE_BLOCKED`                |
| `/patch`    | PLATFORM_DEVOPS | Admin ✅ / Customer ❌ (default) | Admin, Dev               |             2 | Yes (RW)     | Yes (strict)       | patch proposal + diff preview                         | `CONFIRM_REQUIRED` / `READ_ONLY` / `SANDBOX_ONLY` |
| `/hrm`      | PLATFORM_DEVOPS | Admin ✅ / Customer ❌ (default) | Admin, Dev               |             1 | No           | Yes (light)        | cognition surface (safe, high-level)                  | `RB_MODE_REQUIRED` / `SCOPE_BLOCKED`             |
| `/registry` | PLATFORM_DEVOPS | Admin ✅ / Customer ❌         | Admin, Dev               |             1 | Read-only    | Yes (light)        | tool/command registry dump                            | `FORBIDDEN`                                    |
| `/git`      | PLATFORM_DEVOPS | Admin ✅ / Customer ❌         | Admin, Dev               |           2–3 | Yes (RWX)    | Yes (strict)       | repo actions (scoped)                                 | `GEP_REQUIRED` / `CONFIRM_REQUIRED`              |
| `/fs`       | PLATFORM_DEVOPS | Admin ✅ / Customer (read-only) | Admin, Dev, Support      |           1–2 | Yes          | Yes (strict on write) | fs-tree/read/edit                                     | `READ_ONLY` / `FORBIDDEN`                        |

**Notes:**
* Customer scope defaults to **Tier 1 read/help** only.
* Anything that writes must be **Admin-only** unless explicitly enabled later.

**Hard enterprise rule:**
If **Sandbox Required = Yes**, then production BrewAssist **must not execute it directly**. It must return:
* proposal → confirm → sandbox apply route
