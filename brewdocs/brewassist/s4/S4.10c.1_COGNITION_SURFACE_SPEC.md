# S4.10c.1: Identity + HRM "Thinking" Cognition Surface

## Objective
Implement an enterprise-safe “BrewAssist is thinking” cognition surface that shows high-level reasoning signals WITHOUT exposing private chain-of-thought.

## Core Principle
Expose WHAT BrewAssist is considering — never HOW it internally reasons.

## Rules (Non-Negotiable)
- DO NOT expose chain-of-thought or internal deliberation.
- DO NOT show token-by-token reasoning.
- DO NOT add speculative language.
- DO NOT invent new execution capabilities.
- This is a DISPLAY + STATE task, not a logic rewrite.

## Implementation Details

### 1) Cognition State Model
A structured cognition summary object derived from existing systems. This object MUST be deterministic and serializable.

**Fields:**
- `persona`: The selected persona (e.g., "Admin", "Customer").
- `emotionalState`: BrewAssist's current emotional state (e.g., "Neutral", "Cautious").
- `userRole`: The role of the current user.
- `toolbeltTier`: The active Toolbelt Tier (e.g., "T1_SAFE", "T3_DANGEROUS").
- `intent`: The inferred user intent.
- `reasoningMode`: The active reasoning mode (e.g., "LLM", "HRM", "Agent", "Loop").
- `riskLevel`: The assessed risk level (e.g., "Low", "Moderate", "High").
- `executionPermission`: Whether execution is allowed (e.g., "ALLOW", "ALLOW_WITH_WARNING", "REQUIRE_CONFIRMATION", "BLOCK").
- `truthValidationStatus`: The status of BrewTruth validation (e.g., "Validated", "Uncertain", "Failed").

**Example (conceptual only):**
```json
{
  "persona": "Admin",
  "emotionalState": "Neutral",
  "userRole": "Developer",
  "toolbeltTier": "T1_SAFE",
  "intent": "Code Generation",
  "reasoningMode": "LLM",
  "riskLevel": "Low",
  "executionPermission": "ALLOW",
  "truthValidationStatus": "Validated"
}
```

### 2) Cognition Phase Labels (UI-Safe)
Replace generic “BrewAssist is thinking…” with phase-based status lines. Only ONE line visible at a time. No scrolling logs.

**Phase Labels:**
- `Evaluating request…`
- `Interpreting intent and authority…`
- `Checking safety and permissions…`
- `Selecting reasoning strategy…`
- `Preparing response…`

### 3) Identity + HRM Integration
Ensure cognition surface reflects:
- Persona selection (tone alignment)
- HRM vs LLM vs Agent selection
- Toolbelt tier constraints
- BrewTruth requirement (true/false)

### 4) Failure & Uncertainty Handling
If BrewTruth confidence is low or context is missing:
- Cognition surface must indicate uncertainty
- Response must slow down and ask for clarification
- No execution occurs

### 5) Auditability
Cognition state should be:
- Loggable (BrewLast compatible)
- Reviewable by Admin
- Invisible to non-admin users unless explicitly enabled

## Allowed Files

## Validation
- No chain-of-thought leakage
- UI displays cognition clearly and succinctly
- No regression in existing tests
- Admin-only visibility by default

## Deliverable
Enterprise-grade cognition transparency that builds trust without compromising safety.
