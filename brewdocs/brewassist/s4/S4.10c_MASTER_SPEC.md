## S4.10c Master Spec

### Goal

Make **intent + toolbelt tier + BrewTruth** work as one consistent policy layer:

1.  **Intent resolved first** (what the user is trying to do)
2.  **Toolbelt tier resolved next** (what actions are allowed)
3.  **BrewTruth grades the response + flags risk** (how safe/trustworthy the outcome is)
4.  **Handshake decision**: allow / warn / block / require-confirmation

### Acceptance Criteria

You’re “green” on S4.10c when:

*   BrewTruth is **not stubbed** and scores vary by content
*   Flags populate (missing_context / speculative / possible_hallucination / safety_concern / fallback_used)
*   Toolbelt “block” responses are explainable (“why blocked”, “what to do next”)
*   Tests pass:
    *   BrewTruth varies on different responses
    *   High-risk content triggers safety flags and drops tier
    *   Tool run truth check works and adds flags
