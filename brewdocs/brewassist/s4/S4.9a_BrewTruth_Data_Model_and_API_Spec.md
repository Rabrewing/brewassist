## 2. Core Types (TypeScript spec)

> File: `lib/brewtruth.ts`

```ts
// BrewTruth grading tier ‚Äì how "trustworthy / production-ready" a reply is.
export type BrewTruthTier =
  | "gold"        // Safe to trust in most cases
  | "silver"      // Good, but may need quick review
  | "bronze"      // Usable draft, needs editing
  | "red";        // High risk / likely wrong / incomplete

// Dimensions we care about for Tier-2.
export type BrewTruthDimension =
  | "factuality"
  | "relevance"
  | "clarity"
  | "safety"
  | "structure";

// One dimension‚Äôs score.
export interface BrewTruthScore {
  dim: BrewTruthDimension;
  score: number;      // 0‚Äì1 (or 0‚Äì100, see note below)
  notes?: string;     // short human explanation
}

// Flags for quick UI badges and logs.
export type BrewTruthFlagType =
  | "low_confidence"
  | "speculative"
  | "missing_context"
  | "possible_hallucination"
  | "safety_concern"
  | "partial_answer"
  | "tool_failure"
  | "fallback_used";

// Lightweight trace of which model / provider answered.
export interface BrewTruthModelTrace {
  provider: "openai" | "gemini" | "mistral" | "nims" | "tinyllm" | "unknown";
  model: string;              // e.g. "gpt-4.1-mini", "gemini-2.5-flash"
  routeType: "primary" | "fallback" | "research" | "preferred" | "unknown";
  latencyMs?: number;
}

// The main BrewTruth object we expose everywhere.
export interface BrewTruthReport {
  tier: BrewTruthTier;
  overallScore: number;        // 0‚Äì1 (Tier-2) or 0‚Äì100 (if you prefer)
  scores: BrewTruthScore[];    // one per dim
  flags: BrewTruthFlagType[];
  summary: string;             // 1‚Äì2 sentence recap, UI-ready
  modelTrace: BrewTruthModelTrace;
  evaluatedAt: string;         // ISO timestamp
  version: string;             // e.g. "bt-2.0"
}
```

**Design note:**

* For simplicity, I‚Äôd do **0‚Äì1** for scores internally, and let the UI format as `%` if needed.

## 3. BrewTruth Engine Function Contract

> File: `lib/brewtruth.ts`

Add / confirm an exported function with a **stable signature**:

```ts
export interface BrewTruthInput {
  prompt: string;          // what user asked
  response: string;        // what BrewAssist answered
  mode?: BrewMode;         // "llm" | "hrm" | "agent" | "loop"
  providerTrace?: BrewTruthModelTrace; // optional input if engine already knows
}

export async function runBrewTruth(
  input: BrewTruthInput
): Promise<BrewTruthReport> {
  // implementation uses OpenAI (or other) internally
}
```

* Internally you can still use OpenAI/Gemini to ‚Äúself-grade‚Äù; Tier-2 doesn‚Äôt expose that complexity.
* `providerTrace` lets the caller (engine) pass in the active provider/model/routeType instead of re-inferring it.

## 4. API Contracts

### 4.1 `/api/brewassist` (existing, now BrewTruth-aware)

**Request (no change required, just spec‚Äôing):**

```jsonc
POST /api/brewassist
{
  "input": "Explain BrewAssist like I'm mentoring a junior dev.",
  "mode": "llm",
  "useResearchModel": false,
  "modelRole": "llm"
}
```

**Response (new contract fields highlighted):**

```jsonc
200 OK
{
  "message": {
    "role": "assistant",
    "content": "‚Ä¶reply text‚Ä¶"
  },
  "text": "‚Ä¶reply text‚Ä¶",

  // NEW: BrewTruth Tier-2 fields
  "truth": {
    "tier": "silver",
    "overallScore": 0.82,
    "scores": [
      { "dim": "factuality", "score": 0.8,  "notes": "Mostly correct." },
      { "dim": "relevance",  "score": 0.9,  "notes": "Directly answers." },
      { "dim": "clarity",    "score": 0.85, "notes": "Well structured." },
      { "dim": "safety",     "score": 0.95, "notes": "No obvious risks." },
      { "dim": "structure",  "score": 0.7,  "notes": "Could be shorter." }
    ],
    "flags": ["partial_answer"],
    "summary": "Good answer for internal dev use; shorten and tighten before sending to a client.",
    "modelTrace": {
      "provider": "openai",
      "model": "gpt-4.1-mini",
      "routeType": "primary",
      "latencyMs": 1832
    },
    "evaluatedAt": "2025-12-09T15:17:22.123Z",
    "version": "bt-2.0"
  },

  // Existing routing metadata
  "modelRole": "llm",
  "modelRoleUsed": "llm",
  "providerUsed": "openai",
  "modelUsed": "gpt-4.1-mini",
  "routeType": "primary"
}
```

**When BrewTruth is disabled (env flag or error):**

* `truth` may be:

  * Omitted entirely, or
  * Set to `null`.
* Engine **must not throw** if BrewTruth fails:

  * It should log the error and still return the core assistant reply.

I‚Äôd recommend a simple env flag:

```bash
# .env.local
BREWTRUTH_ENABLED=true
BREWTRUTH_TIER=2
```

And in code:

```ts
const truth =
  process.env.BREWTRUTH_ENABLED === "true"
    ? await runBrewTruth(input)
    : undefined;
```

### 4.2 `/api/brewtruth` (optional direct endpoint)

> Only if you want a direct grading endpoint for the UI / MCPTools.

**Request:**

```jsonc
POST /api/brewtruth
{
  "prompt": "User question here‚Ä¶",
  "response": "Assistant reply here‚Ä¶",
  "mode": "llm",
  "providerTrace": {
    "provider": "openai",
    "model": "gpt-4.1-mini",
    "routeType": "primary",
    "latencyMs": 950
  }
}
```

**Response:**

```jsonc
200 OK
{
  "truth": { /* BrewTruthReport as defined above */ }
}
```

**Error behavior:**

* On internal error (grading failed, upstream model down):

```jsonc
500
{
  "error": "BrewTruth failed to grade this response.",
  "details": "‚Ä¶short, non-sensitive error text‚Ä¶"
}
```

* But `/api/brewassist` should **swallow** that and not expose 500 to the user; this endpoint is more for tools and internal UI.
