## 1. Objective

Turn the Toolbelt Tier + Mode system into a **live cockpit governor**:

* Any change to **Mode** (HRM / LLM / Agent / Loop)
* Any change to **Tier** (T1 / T2 / T3)

…must **instantly**:

1. Update UI (buttons, tooltips, warnings)
2. Update allowed MCP actions
3. Update BrewAssist engine safety rules
4. Update BrewTruth thresholds
5. Be logged for BrewLast / future BrewOps

No refresh, no extra click — **real-time, global, and consistent**.

export type ToolPermission =
  | 'allowed'
  | 'needs_confirm'
  | 'admin_only'

```ts
export interface ToolbeltState {
  mode: BrewMode;
  tier: ToolbeltTier;

```

### 3.6 `ToolbeltRulesSnapshot`

```ts
export interface ToolbeltRulesSnapshot {
  // MCP tool ids
  mcp: Record<string, ToolPermission>;
  // generic actions
  actions: {
    fileWrite: ToolPermission;
    fileDelete: ToolPermission;
    gitCommit: ToolPermission;
    dbMigrate: ToolPermission;
    agentExec: ToolPermission;
  };
  // BrewTruth
  truth: {
    minScoreForWrite: number; // 0–1
    minScoreForSystemChange: number;
  };
}
```

## 4. Core Logic

### 4.1 `computeToolbeltRules(mode, tier)`

New pure function in `lib/toolbeltConfig.ts`:

```ts
export function computeToolbeltRules(
  mode: BrewMode,
  tier: ToolbeltTier
): ToolbeltRulesSnapshot { /* deterministic rules */ }
```

**Ruleset (v1):**

1. **HRM Mode**

* Tier always coerced to **T1_SAFE** internally
* All write/system actions → `blocked`
* MCP File/Git/DB → `blocked`
* Research/NIMs → `allowed`
* `minScoreForWrite` → 1.0 (effectively impossible)

2. **LLM Mode**

* **T1_SAFE**

  * fileWrite/fileDelete/gitCommit/dbMigrate/agentExec → `blocked`
  * Research/NIMs → `allowed`
  * `minScoreForWrite` → 0.9
* **T2_GUIDED**

  * fileWrite → `needs_confirm`
  * gitCommit → `needs_confirm`
  * dbMigrate → `blocked`
  * agentExec → `needs_confirm`
  * `minScoreForWrite` → 0.7
* **T3_POWER**

  * fileWrite → `needs_confirm`
  * gitCommit → `needs_confirm`
  * agentExec → `admin_only`
  * dbMigrate → `admin_only`
  * `minScoreForWrite` → 0.7
  * `minScoreForSystemChange` → 0.7

3. **AGENT Mode**

* **T1_SAFE**

  * agentExec → `needs_confirm`
  * all writes → `blocked`
* **T2_GUIDED**

  * fileWrite/fileDelete/gitCommit/dbMigrate/agentExec → `needs_confirm`
* **T3_POWER**

  * fileWrite/fileDelete/gitCommit → `needs_confirm`
  * agentExec/dbMigrate → `admin_only`
  * `minScoreForSystemChange` → 0.7

4. **LOOP Mode**

* Always **T1_SAFE**
* Everything write/system → `blocked`
* Research/NIMs → `allowed`
* agentExec → `blocked`

Implementation note: rules should be **table-driven** so we can tweak without touching engine code.

## 5. State & Context Wiring

### 5.1 `ToolbeltProvider`

Create `contexts/ToolbeltContext.tsx`:

```ts
interface ToolbeltContextValue extends ToolbeltState {
  setMode: (mode: BrewMode) => void;
  setTier: (tier: ToolbeltTier) => void;
}

const ToolbeltContext = createContext<ToolbeltContextValue | null>(null);
```

* Initialize from:

  * defaults: `mode: 'LLM'`, `tier: 'T2_GUIDED'`
  * optional: BrewLast (later S5.0)
* On any `setMode` or `setTier`:

  * recompute `effectiveRules = computeToolbeltRules(mode, tier)`
  * update `lastUpdatedAt`
  * call `logToolbeltEvent(...)` (see §7)

Wrap the main cockpit shell (e.g. `pages/index.tsx` or `BrewCockpitCenter`) in `<ToolbeltProvider>`.

### 5.2 HUD Integration

* Tier dropdown reads from context:

  * `value={tier}`
  * `onChange={setTier}`
* Mode buttons:

  * still set underlying BrewMode
  * must now call `setMode()` on context

## 6. UI Enforcement

### 6.1 MCP Tools (Left Rail)

Where MCP button configs live (e.g. `ActionMenu.tsx` / `WorkspaceSidebarLeft.tsx`):

* Each tool gets:

  * `id` (e.g. `"mcp.file"`, `"mcp.git"`, `"mcp.db"`, `"mcp.research"`)
  * `riskLevel`

Use `ToolbeltContext`:

```ts
const { effectiveRules } = useToolbelt();

const permission = effectiveRules.mcp[tool.id] ?? 'blocked';
```

UI behavior:

* `blocked` →

  * button disabled
  * tooltip: “Disabled in current Mode / Tier”
* `needs_confirm` →

  * show small “⚠ Confirm” label
  * on click: open confirm modal (“Tier 2 patch — apply?”)
* `admin_only` →

  * show lock icon
  * tooltip: “Admin-only in this configuration”
* `allowed` → normal

### 6.2 ActionMenu

* Action presets tagged with `riskLevel`
* Before executing:

  * look up `effectiveRules.actions[<mapped risk>]`
  * same permission handling as above

## 7. Engine Enforcement

### 7.1 New “Toolbelt Guard” Helper

Add `lib/toolbeltGuard.ts`:

```ts
export function isWriteActionAllowed(
  rules: ToolbeltRulesSnapshot,
  risk: RiskLevel
): ToolPermission { /* map risk → permission */ }
```

Use this inside:

* `lib/brewassist-engine.ts`
* MCP execution wrappers
* Agent pipelines

Behavior:

* If `blocked` →

  * abort action
  * return BrewAssist message:
    “This action is blocked in the current Mode/Tier (HRM + Tier 1). Switch to a higher tier or use a different mode.”
* If `needs_confirm` →

  * require explicit confirmation flag from UI
  * if missing → return
    “This is a Tier 2 action. Please confirm before I proceed.”
* If `admin_only` →

  * check future Admin mode flag (stub for now)
  * if not admin → block

### 7.2 BrewTruth Thresholds

In the point where you already call BrewTruth (S4.9c):

* Read `rules.truth.minScoreForWrite` / `minScoreForSystemChange`
* If truth score < threshold:

  * block action
  * respond:
    “BrewTruth score is too low for this kind of change at the current Tier.”
