## S4.9b.env — BrewTruth Environment Variable Hardening
Status: Open (post-S5.x hardening task)

### Objective
To properly and robustly wire the `BREWTRUTH_ENABLED` environment variable (and other `BREWTRUTH` related variables) within the Next.js application, ensuring they are correctly loaded and accessible on the server-side without relying on direct script modifications in `package.json` or temporary development hacks. This task aims to resolve the underlying issue of `process.env.BREWTRUTH_ENABLED` being `undefined` in the Next.js development server environment.

### Background
During the implementation of S4.9b.x, it was discovered that `process.env.BREWTRUTH_ENABLED` was consistently `undefined` within the Next.js application's server-side code, despite being present in `.env.local` and explicitly added to the `env` object in `next.config.js`. This necessitated a temporary workaround of setting `BREWTRUTH_ENABLED=true` directly in the `dev` script of `package.json`. While this workaround allowed for the completion and verification of S4.9b.x, it is not the ideal long-term solution.

### Deliverables

1.  **Root Cause Analysis:**
    *   Investigate why Next.js (potentially in conjunction with Turbopack or the `output: 'standalone'` configuration) is failing to load server-side environment variables from `.env.local` and `next.config.js` during development.
    *   Consult Next.js documentation, community forums, and GitHub issues for known problems and recommended solutions for environment variable loading in similar setups.

2.  **Implement Robust Environment Variable Loading:**
    *   Identify and implement the correct Next.js-idiomatic method for loading server-side environment variables from `.env.local` such that `process.env.BREWTRUTH_ENABLED` (and other `BREWTRUTH` variables) are correctly populated within the application's server-side runtime.
    *   This might involve:
        *   Adjusting `next.config.js` configuration.
        *   Ensuring proper `.env` file placement and naming conventions.
        *   Considering alternative environment variable loading strategies if Next.js's built-in methods prove insufficient for this specific project setup.

3.  **Remove `package.json` Script Modification:**
    *   Once the robust environment variable loading is implemented and verified, revert the `dev` script in `package.json` to its original state (e.g., `"dev": "next dev -p 3000"`), removing the explicit `BREWTRUTH_ENABLED=true` prefix.

4.  **Verification:**
    *   Ensure that `BREWTRUTH_ENABLED` (and other `BREWTRUTH` variables) are correctly accessible within the Next.js application's server-side code (e.g., in API routes) without the `package.json` script modification.
    *   Re-run the curl tests for S4.9b.x (specifically TEST 2 and TEST 3) to confirm that `BrewTruth` is enabled/disabled as expected based on the `.env.local` configuration.
    *   Ensure the existing Jest test suite (`__tests__/api/brewtruth.test.ts`) continues to pass.
