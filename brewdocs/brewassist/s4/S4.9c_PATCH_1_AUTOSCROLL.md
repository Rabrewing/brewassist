## S4.9c — Implementation Patch #1 (Auto-Scroll & Input UX)

### Objective

Bring the BrewAssist chat log behavior closer to ChatGPT’s runtime UX:

- New messages should scroll into view automatically.
- Manual scrolling should be respected (no “yank back to bottom” while the user is reading).
- The BrewTruth chip and formatting toolbar should feel visually aligned and not “sitting on the floor” of the input bar.

---

### Scope

1. **Chat Log Auto-Scroll**
2. **Scroll Lock When User Scrolls Up**
3. **Input Bar Layout Polish (TruthChip + Toolbar)**

---

### 1. Chat Log Auto-Scroll

**Behavior:**

- When a new assistant or user message is appended:
  - If the user is **already at (or near) the bottom**, the log **auto-scrolls** to the newest message.
  - If the user has scrolled up, **do not** auto-scroll.

**Implementation Notes (React-style):**

- Add a `ref` to the scrollable chat container, e.g. `chatContainerRef`.
- Track an `autoScrollEnabled` boolean in state.

- On `messages` change:

  ```ts
  const isNearBottom =
    chatContainer.scrollHeight - (chatContainer.scrollTop + chatContainer.clientHeight) < 80;

  if (autoScrollEnabled && isNearBottom) {
    chatContainer.scrollTo({ top: chatContainer.scrollHeight, behavior: "smooth" });
  }
  ```

* Attach an `onScroll` handler:

  ```ts
  const distanceFromBottom =
    chatContainer.scrollHeight - (chatContainer.scrollTop + chatContainer.clientHeight);

  if (distanceFromBottom > 120) {
    setAutoScrollEnabled(false); // user reading history
  } else {
    setAutoScrollEnabled(true);  // user came back to bottom
  }
  ```

**Acceptance Criteria:**

* New responses scroll into view automatically when the user is at the bottom.
* If the user scrolls up, new responses do **not** yank the view.
* When the user scrolls back near the bottom, auto-scroll re-enables.

---

### 2. “Thinking” Thumb-Through Effect

Tie the existing “BrewAssist is thinking” indicator to the new scroll logic:

* When a user hits **Enter**:

  * Lock the chat view to bottom and show a subtle “processing” shimmer / waveform.
* When the assistant response arrives:

  * Auto-scroll the final answer into view (if `autoScrollEnabled` is true).

No new visuals required yet—just ensure the **existing indicator** appears immediately on send and disappears when the response finishes.

---

### 3. Input Bar Layout Polish (TruthChip + Toolbar)

**Goals:**

* TruthChip pill sits **centered vertically** in the input row.
* Formatting toolbar (B, I, H1, H2, `</>`) feels docked to the top edge of the input, not floating too low.
* No overlap with the text caret.

**Implementation Notes:**

* Wrap the toolbar + input + TruthChip in a flex row:

  ```tsx
  <div className="brew-input-row">
    <div className="brew-markdown-toolbar">…</div>
    <textarea … />
    <TruthChip … />
  </div>
  ```

* Use CSS:

  ```css
  .brew-input-row {
    display: flex;
    align-items: center;        /* vertically center chip + input */
    gap: 0.5rem;
  }

  .brew-truth-chip {
    align-self: center;
    margin-bottom: 0;          /* remove “floor hugging” */
  }
  ```

**Acceptance Criteria:**

* TruthChip appears as a clean pill, roughly centered vertically with the input text.
* Toolbar icons no longer overlap or sit misaligned relative to the input.
* No clipping or overlap at different viewport widths.
