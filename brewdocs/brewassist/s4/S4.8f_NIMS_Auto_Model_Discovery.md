**Status:** Approved for implementation

## **1. Overview**

NVIDIA NIMs will serve as BrewAssist’s **Researcher-tier provider**.
However, NVIDIA exposes **different models to different API keys**, and unavailable models return a **404**.

To prevent runtime failures, S4.8f introduces:

* **NIMs Auto-Model Discovery**
* **Multi-fallback Researcher Routing**
* **Env-controlled enable/disable of NIMs**
* **Graceful fallback to Gemini/OpenAI when NIMs is unavailable**
* **Probe-first selection** to avoid 404 / 401 loops
* **Researcher metadata tagging** (for BrewTruth & BrewLast)

This ensures BrewAssist always chooses a working NIM model — or falls back safely.

## **4. pickNimsModel() — Auto-Discovery Logic**

### **4.1 Function Behavior**

```ts
async function pickNimsModel() {
  const models = [
    process.env.NIMS_MODEL_PREFERRED,
    process.env.NIMS_MODEL_FALLBACK_1,
    process.env.NIMS_MODEL_FALLBACK_2,
  ].filter(Boolean);

  for (const model of models) {
    const ok = await probeNimsModel(model);
    if (ok) return model;
  }

  return null; // NIMs unavailable
}
```

---

## **5. probeNimsModel() — 1-Token Health Check**

### **5.1 Purpose**

NVIDIA returns **404** for:

* wrong model name
* models not accessible to your API key
* models not enabled in your region
* expired keys

We avoid failures by probing with a minimal request:

```ts
async function probeNimsModel(model: string): Promise<boolean> {
  const res = await fetch(`${NIMS_BASE_URL}/chat/completions`, {
    method: "POST",
    headers: {
      "Authorization": `Bearer ${NIMS_API_KEY}`,
      "Content-Type": "application/json"
    },
    body: JSON.stringify({
      model,
      messages: [{ role: "user", content: "ping" }],
      max_tokens: 1
    }),
  });

  return res.ok;
}
```

If `res.status !== 200` → model is rejected silently.

## **6. brewassist-engine.ts Changes**

### **6.1 When research mode is selected**

```ts
if (routeType === "research" && process.env.NIMS_ENABLED === "true") {
  const selectedModel = await pickNimsModel();
  
  if (selectedModel) {
    return callNimsProvider(selectedModel, inputMessages, opts);
  }
}
```

### **6.2 If all models fail**

Fallback automatically to:

1. Gemini Reasoning
2. OpenAI Deep (gpt-4.1)
3. Mistral (if configured)

This ensures *no user workflow is ever blocked.*

## **7. Provider Call for NIMs**

NIMs uses the **OpenAI-style schema**:

```ts
async function callNimsProvider(model, messages, opts) {
  const res = await fetch(`${NIMS_BASE_URL}/chat/completions`, {
    method: "POST",
    headers: {
      "Authorization": `Bearer ${NIMS_API_KEY}`,
      "Content-Type": "application/json",
    },
    body: JSON.stringify({
      model,
      messages,
      max_tokens: Number(process.env.NIMS_MAX_TOKENS ?? 1536),
      temperature: Number(process.env.NIMS_TEMPERATURE ?? 0.2),
    }),
  });

  const data = await res.json();
  return data?.choices?.[0]?.message?.content ?? null;
}
```
