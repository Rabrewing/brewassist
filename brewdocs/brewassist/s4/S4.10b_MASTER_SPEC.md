## S4.10b Master Spec

### Goal

Visual + UX refinement of the cockpit while preserving all S4.10 enforcement (admin/customer split, proposal-only customer toolbelt, sandbox admin-only, BrewTruth silent for customers).

### Scope

1.  **Scroll + Layout Stability**
    *   Only the center chat pane scrolls (not header/footer)
    *   Side panes stay fixed (left MCP tools, right file tree/sandbox)
    *   Scrollbars visible + consistent (no “hidden” scroll)

2.  **Spacing + Alignment**
    *   Message card spacing more readable
    *   Input bar aligned, consistent padding/margins
    *   Footer doesn’t overlap the middle pane
    *   Dock/Mode chips aligned and not cramped

3.  **Visibility + Mode Clarity**
    *   Customer mode badge stays visible
    *   Admin mode badge stays visible
    *   BrewTruth chip: admin visible, customer hidden (do not change behavior)

4.  **Polish**
    *   Typography consistency (no “crazy font” issues)
    *   Buttons consistent sizing
    *   No layout jitter when toggling modes

### Non-Goals (must NOT change)

*   No changes to execution gating
*   No changes to toolbelt proposal/execute rules
*   No changes to routing/provider logic
*   No changes to sandbox permission rules
*   No changes to chain/test gates semantics

### Acceptance Criteria

You’re “green” on S4.10b when:

*   Customer mode: clean, stable UI; MCP tools visible; no sandbox; no execution controls
*   Admin mode: full cockpit; sandbox visible; no overlap; everything aligned
*   Scroll behavior: center only; header/footer fixed; scrollbar visible
*   `pnpm test:chain` still green (G1–G9)
*   `pnpm lint` and `pnpm build` green
