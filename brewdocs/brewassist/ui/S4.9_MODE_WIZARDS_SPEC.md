# S4.9 - Per-Mode Wizards Specification

## Goal
Implement conversational wizards for each BrewAssist mode (HRM, LLM, Agent, Loop) to guide the user and tune BrewAssist's behavior for specific tasks within that mode. These wizards are intended to run after the `/init` wizard.

## Per-Mode Wizards (after /init)

Once `/init` completes, BrewAssist can run short mode wizards:

### 1. `/hrm` Wizard – Strategy & Teaching
*   Ask: “Do you want a roadmap, an explanation, or a teaching session?”
*   Options:
    *   **Roadmap** – Sprints, phases, tasks.
    *   **Explanation** – e.g. “Explain this stack and risks.”
    *   **Teaching** – step-by-step, with quizzes.
*   Sets `mode="hrm"` and specialized system text:
    > “You are in HRM mode – focus on structure, clarity, and mentoring RB.”

### 2. `/llm` Wizard – Direct Help
*   Ask: “Are we doing code, docs, or architecture Q&A?”
*   Options:
    *   Code help (snippets, debugging)
    *   Docs (writing, refactor, READMEs)
    *   Architecture questions
*   Sets `mode="llm"` and tunes for **shorter, direct answers**.

### 3. `/agent` Wizard – Tool-driven Ops
*   Ask: “What kind of action do you want to take?”
    *   Modify files
    *   Run commands (lint/build/tests)
    *   Generate patches
    *   Sandbox only vs real repo
*   Sets `mode="agent"` and instructs BrewAssist to suggest tool calls and patch flows.

### 4. `/loop` Wizard – Recap & Improvement
*   Ask: “What should I loop on?”
    *   Last N messages
    *   A file or component
    *   A failing test
*   Sets `mode="loop"` and focuses on:
    *   Recap
    *   Error analysis
    *   Improvement suggestions
    *   Next best action

## API & Session Hook
*   `/init` results update `sessionContext` (skillLevel, projectName, etc.).
*   Mode wizards just **override `mode` and adjust a small config block**.
*   All of this still flows through the same `/api/brewassist` endpoint.
