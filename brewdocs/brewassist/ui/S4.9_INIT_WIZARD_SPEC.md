# S4.9 - `/init` Wizard – Global Onboarding Specification

## Goal

Implement a structured onboarding wizard for the BrewAssist cockpit, triggered by `/init`, to configure the user's environment, repo/provider binding, and work goals.

## Trigger Options

- Type `/init` in the input bar
- Click a “Init” / “Setup Wizard” control (future)
- First-time session auto-prompt:
  > “Do you want to run /init to set up your environment?”
- First-run onboarding banner in the center pane with a direct setup action
- slash-command palette for power users who prefer command-driven setup

## Wizard Steps

1.  **Identify user type**
    - Vibe coder / self-taught
    - Intermediate developer
    - Senior dev
    - Enterprise / architect
    - Non-technical
      → This sets `skillLevel` in `sessionContext` and informs the first recommended mode.

2.  **Determine the work goal**
    - Create a new app
    - Connect to an existing repo
    - Fix a broken environment
    - Generate or refine a roadmap
    - Learn / study (training)
    - Something else (free text)

3.  **Environment choice**
    - New environment
    - Existing repo
    - Not sure yet

4.  **If New environment** – ask:
    - Frontend framework
    - Backend language
    - CSS system
    - Database type
    - Hosting preference
    - Any existing docs/roadmap? (invite upload later)

5.  **If Existing repo** – ask:
    - Where is it hosted? (GitHub/GitLab/Bitbucket/local)
    - Which provider should BrewAssist bind to? (GitHub/GitLab/Bitbucket/local)
    - Monorepo? Y/N
    - Package manager (pnpm/npm/yarn)
    - Can BrewAssist assume `pnpm lint && pnpm build` or different commands?

6.  **Provider and repo binding**
    - Reuse the same `RepoProviderSelector` surface that the center pane uses
    - Capture repo provider and repo root in the shared connection context
    - Explain that all reads and previews happen against the sandbox mirror

7.  **Enterprise setup (when relevant)**
    - Org name
    - Workspace name
    - Initial role
    - Whether team collaboration and reporting should be enabled now

8.  **Summarize + Save**
    - BrewAssist prints an **“Understanding Report”**:
      - Who you are
      - What you’re doing
      - Mode recommendation (HRM/LLM/Agent/Loop)
      - Next 3 suggested actions
      - Suggested first prompt for the center pane
    - On completion, the wizard should prefill the center prompt with the next recommended step

This now renders as a modal wizard in the center flow and remains triggerable from `/init`.
