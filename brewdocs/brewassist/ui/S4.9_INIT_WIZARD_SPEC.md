# S4.9 - `/init` Wizard – Global Onboarding Specification

## Goal

Implement a structured onboarding wizard for the BrewAssist cockpit, triggered by `/init`, to configure the user's environment and work goals.

## Trigger Options

- Type `/init` in the input bar
- Click a “Init” / “Setup Wizard” control (future)
- First-time session auto-prompt:
  > “Do you want to run /init to set up your environment?”

## Wizard Steps (text-only v1)

1.  **Identify user type**
    - Vibe coder / self-taught
    - Intermediate developer
    - Senior dev
    - Enterprise / architect
    - Non-technical
      → This sets `skillLevel` in `sessionContext`.

2.  **Determine the work goal**
    - Create a new app
    - Connect to an existing repo
    - Fix a broken environment
    - Generate or refine a roadmap
    - Learn / study (training)
    - Something else (free text)

3.  **Environment choice**
    - New environment
    - Existing repo
    - Not sure yet

4.  **If New environment** – ask:
    - Frontend framework
    - Backend language
    - CSS system
    - Database type
    - Hosting preference
    - Any existing docs/roadmap? (invite upload later)

5.  **If Existing repo** – ask:
    - Where is it hosted? (GitHub/GitLab/Bitbucket/local)
    - Which provider should BrewAssist bind to? (GitHub/GitLab/Bitbucket/local)
    - Monorepo? Y/N
    - Package manager (pnpm/npm/yarn)
    - Can BrewAssist assume `pnpm lint && pnpm build` or different commands?

6.  **Summarize + Save**
    - BrewAssist prints an **“Understanding Report”**:
      - Who you are
      - What you’re doing
      - Mode recommendation (HRM/LLM/Agent/Loop)
      - Next 3 suggested actions

This is **all conversational** in S4.9 v1 – no UI modals yet.
