# S4.3 – BrewTruth Engine: Mode-Aware Integration (Hard / Soft / RB)
 
## 0️⃣ Objective
 
Integrate safety modes into BrewAssist using the BrewTruth Engine. Modes determine how BrewAssist behaves when an action is flagged as HIGH risk.
 
Modes:
 
* **hard** – Full blocking, requires manual override.
* **soft** – Warns and requires confirmation.
* **rb** – Warns once, then auto‑proceeds on repeated request.
 
All logic is local (no DB tables yet).

## 4️⃣ Mode-Aware Gating Logic
 
### 4.1 Decision Function
 
```ts
export function decideFromTruth(mode, truthScore, riskLevel, summary) {
  if (riskLevel === "LOW" || riskLevel === "MEDIUM") {
    return { action: "proceed", message: summary };
  }
 
  switch (mode) {
    case "hard":
      return {
        action: "block",
        message: `⚠️ HIGH risk. Hard mode blocks this. Summary: ${summary}`,
      };
 
    case "soft":
      return {
        action: "confirm",
        message: `⚠️ HIGH risk. Confirm with “go ahead” to continue. Summary: ${summary}`,
      };
 
    case "rb":
    default:
      return {
        action: "confirm",
        message: `⚠️ HIGH risk detected. RB Mode: say again or confirm to auto‑proceed. Summary: ${summary}`,
      };
  }
}
```

## 5️⃣ Integrating With BrewAssist
 
### 5.1 Risk Flow
 
1. User sends request.
2. If not risky → normal BrewAssist.
3. If risky → evaluate via BrewTruth.
4. Pass result to `decideFromTruth`.
5. BrewAssist follow-up depends on mode.
 
### 5.2 RB Auto‑Proceed Memory
 
```ts
const memory = new Map();
 
export function recordHighRiskWarning(userId, prompt) {
  memory.set(userId, {
    lastPromptHash: hashPrompt(prompt),
    warnedAt: Date.now(),
  });
}
 
export function shouldAutoProceedAfterWarning(userId, prompt) {
  const state = memory.get(userId);
  if (!state) return false;
  return (
    state.lastPromptHash === hashPrompt(prompt) &&
    Date.now() - state.warnedAt < 300000
  );
}
```
