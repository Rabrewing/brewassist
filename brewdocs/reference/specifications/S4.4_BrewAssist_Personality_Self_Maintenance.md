**Phase:** S4.4 – Risk-Aware Personality & Self-Maintenance

## 1. Objective

This phase integrates the Risk-Aware Personality & Conversation Layer with the Self-Repair / Maintenance Engine, transforming BrewAssist into a self-monitoring, self-improving, emotion-aware, safety-aware, and context-persistent AI DevOps assistant. It also introduces a dual-repo architecture for safe AI DevOps in a dedicated sandbox.

---

## 2. Implementation Details

### 2.1 BrewAssist Personality Layer (`lib/brewassistPersonality.ts`)

This module provides lightweight, stateless helpers to shape BrewAssist's tone and RB-aware behavior.
*   **`BrewRiskMode`**: Defines risk gating modes ("HARD", "SOFT", "RB").
*   **`BrewAssistPersonalityConfig`**: Configuration for personality, including `devName`, `mode`, and `maxContextMessages`.
*   **`BrewAssistContextMessage`**: Interface for recent messages to maintain conversational flow.
*   **`initPersonalityState()`**: Initializes the personality state.
*   **`inferTone()`**: A heuristic to determine the conversational tone based on the user prompt and last tool summary.
*   **`applyPersonalityLayer()`**: The core function that injects RB-aware framing, adds a short context banner, and produces a `toneHint` for the downstream risk engine. It also updates the rolling context of recent messages.

### 2.2 BrewAssist Risk Engine (`lib/brewassistRiskEngine.ts`)

This module handles risk-aware routing and stop/override logic.
*   **`RiskLevel`**: Defines risk levels ("LOW", "MEDIUM", "HIGH").
*   **`RiskEvaluation`**: Details the risk level, reasons, and suggested action (ALLOW, WARN, BLOCK).
*   **`RiskContext`**: Provides context for risk evaluation, including `mode`, `toneHint`, `userPrompt`, and `toolName`.
*   **`RiskDecision`**: The final decision on whether to allow, require override, or block an action.
*   **`evaluateRisk()`**: A heuristic to assess the risk level of a user prompt based on dangerous keywords and tone.
*   **`decideRisk()`**: Converts the risk evaluation and active `BrewRiskMode` into a concrete gating decision, generating appropriate messages for the user.

### 2.3 BrewAssist Self-Maintenance Engine (`lib/brewassistMaintenance.ts`)

This module provides a skeleton for internal diagnostics and self-repair.
*   **`HealthCheckResult`**: Reports the health status, warnings, errors, and details.
*   **`MaintenanceSuggestion`**: Describes a suggested fix, its kind, description, and target path.
*   **`BREWEXEC_ROOT`**: Defines the root path of the BrewExec repo.
*   **`sandboxPath()`**: Helper for resolving paths within the `sandbox/` directory.
*   **`runBrewAssistHealthChecks()`**: Performs basic health checks on essential files and the `sandbox/` directory.
*   **`generateMaintenanceSuggestions()`**: Generates maintenance suggestions, such as creating the `sandbox/` directory if it doesn't exist.

### 2.4 New API Endpoints

*   **`pages/api/brewassist-health.ts`**: A GET endpoint to expose the results of `runBrewAssistHealthChecks()`, providing a health snapshot of BrewAssist.
*   **`pages/api/brewassist-suggest.ts`**: A GET endpoint to return maintenance suggestions generated by `generateMaintenanceSuggestions()`.
*   **`pages/api/brewassist-sandbox-apply.ts`**: A POST endpoint for safely writing files exclusively within the `sandbox/` directory, preventing any accidental modifications to production code. It includes path validation to ensure files do not escape the sandbox.

### 2.5 `sandbox/` Directory

The `sandbox/` directory was created as an isolated environment for all AI-generated code, patch plans, and maintenance metadata. This ensures that BrewAssist's self-repair and code generation activities are contained and do not affect the production codebase without explicit RB approval.

### 2.6 Patch `pages/api/llm-tool-call.ts`

The `llm-tool-call.ts` handler was patched to integrate the personality and risk pipeline:
1.  **Imports**: Added necessary imports for `brewassistPersonality`, `brewassistRiskEngine`, and `brewModeServer`.
2.  **Personality and Risk Integration**:
    *   Initializes `personalityState` using `initPersonalityState` and `getUserMode`.
    *   Applies the `personalityLayer` using `applyPersonalityLayer` to shape the prompt and infer tone.
    *   Evaluates risk using `decideRisk` based on the personality's mode and tone.
    *   If `riskDecision.allow` is `false`, the tool execution is blocked, a message is returned to the user, and the blocked attempt is logged to BrewLast.
    *   If allowed, the `personality.promptForModel` (the prompt shaped by the personality layer) is used for the actual LLM/tool invocation.
    *   The `userPrompt` and `userId` are passed from the `ToolRequestBody` to the personality and risk evaluation.
3.  **Response Enrichment**: The API response now includes `personalityMeta` and `riskDecision` for enhanced context.

### 2.7 Updated `lib/openaiToolbelt.ts`

*   The `callToolbeltApi` function was modified to accept `userPrompt` and `userId` as arguments and pass them in the request body to the `/api/llm-tool-call` endpoint.
*   The `runWithToolbelt` function was updated to pass the `prompt` (as `userPrompt`) and `options.userId` to `callToolbeltApi`.
