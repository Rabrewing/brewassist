# S4.10 — MCP DevOps Cockpit Blueprint (Next Big Step)

## Goal
Turn the right side + MCP tools into a **DevOps control cockpit** that supports:

* Small vibe-coder teams
* Enterprise squads
* Live collaboration between “Dev” and “Ops” inside BrewAssist

## 1️⃣ Right Pane Tab Model
Right pane becomes **tabbed**, with these tabs (for example):

1. **Files** — repository tree (current behavior, improved)
2. **Collab** — team chat / video / notes
3. **History** — chat history + search + pinning
4. **Ops** — tests, builds, deployments status
5. **Docs** — context docs, architecture notes, BrewDocs integration

Each tab = its own “surface” that still shares the same project context.

## 2️⃣ Files Tab (Dev Side)
* Show repo tree, filtered by root (e.g. `/brewassist`).
* Clicking a file:

  * Opens it in center panel (or previews).
  * Context is available to MCP Tools and BrewAssist prompts.
* Supports:

  * File search
  * Type filters
  * “Show only changed files” view (hook into git status later).

## 3️⃣ Collab Tab (People Side)
* Side chat panel for team members (not the AI chat).
* Features:

  * Real-time notes
  * “Assign to” tags
  * Simple presence indicator (who is “in this cockpit”).

Long term:

* Embed audio/video (e.g. Twilio, Daily, or simple iframe integration).
* “Follow mode” where one user’s scroll/selection is mirrored.

## 4️⃣ History Tab
* Shows **past BrewAssist conversations** within this project.
* Search by:

  * Keywords
  * Dates
  * Truth tier (e.g. “show all Red/low-truth answers we should revisit”).
* Each past interaction:

  * Click → re-open in the center workpane
  * Option → “Re-grade with latest models”

## 5️⃣ Ops Tab
* Surface for:

  * `pnpm lint`
  * `pnpm test`
  * `pnpm build`
  * Deployment hooks (Vercel, etc.)
* MVP:

  * Show last run command + exit code + logs.
  * Trigger new runs via MCP Tools.
  * Mark success/failure with icon & timestamp.

## 6️⃣ Docs Tab
* Hooks into `/brewdocs`.
* Tabs within Docs:

  * Roadmaps
  * Specs
  * Case studies
  * Architecture diagrams (later embed images).
* Click any doc:

  * Preview text in the right pane.
  * “Send to BrewAssist” button to prefill context into the center.

## 7️⃣ S4.10 Execution Plan (High-Level)
You can treat S4.10 as:

* **S4.10a** — Right-pane Tab Infrastructure (UI only)
* **S4.10b** — Files Tab upgrades
* **S4.10c** — History + Docs skeleton
* **S4.10d** — Ops tab skeleton
* **S4.10e** — Collab tab skeleton
* **S5.x** — Deep behavior (real git wiring, BrewDocs search, etc.)
