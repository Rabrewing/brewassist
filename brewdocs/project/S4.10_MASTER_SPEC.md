# S4.10 — MASTER SPEC
## Admin Mode vs Customer Mode Separation
### BrewAssist Cockpit Access Control & UI Partitioning

**THIS DOCUMENT IS THE AUTHORITATIVE AND UNIFIED MASTER SPECIFICATION FOR S4.10.**
**It supersedes all previous drafts, including S4.10a, S4.10b, and any sandbox specification fragments.**
**All future implementation and documentation for S4.10 must refer to this document as the single source of truth.**

**1. Purpose**
S4.10 introduces the formal split of BrewAssist into two operating modes:
🔶 **Admin Mode** (Internal / RB-only / Developer Access)
Full cockpit, sandbox, wizards, MCP tools, chain engines, BrewTruth, fallbacks, and toolbelt tiers.
🔷 **Customer Mode** (External User Access)
Clean, minimal UI with no access to internal DevOps systems.
This is required for:
*   Security
*   Preventing customers from seeing Dev tools
*   Preparing BrewAssist to be a SaaS product
*   Enforcing future per-user permissions
*   Laying the foundation for role-based onboarding and pricing tiers

**2. Mode Definition**
**2.1 Admin Mode**
Full unrestricted DevOps cockpit.
Admin Mode unlocks:
*   Sandbox panel
*   MCP Tools in sidebar
*   Action Menu (full version)
*   Toolbelt Tier 1 → Tier 3
*   Chain engine (HRM / LLM / Agent / Loop)
*   /init Wizard
*   Repo access (future S5.x)
*   Multi-AI fallback chain
*   BrewTruth pre-execution checks
*   Gatekeeper override capabilities
*   Dev logs
*   BrewLast session persistence
*   System debug surfaces

Admin Mode Behavior:
*   UI = Full cockpit
*   Colors = Deep Black, BrewGold, BrewTeal (default theme)
*   Risk labels visible
*   TruthChip always shown
*   Context chips shown
*   Advanced keyboards shortcuts enabled

**2.2 Customer Mode**
Minimal BrewAssist interface.
Customer Mode hides:
*   Sandbox
*   File tree previews
*   MCP Tools (ALL tiers)
*   Action Menu
*   Wizard system
*   Folder visibility
*   Repo interactions
*   Dev chains
*   BrewTruth toggle

Customer Mode UI:
*   Clean right pane
*   Only AI assistant stream + formatting toolbar
*   No dev logs
*   No debugging
*   No toolbelt actions
*   No internal context metadata

**3. Routing Rules**
**3.1 Mode determined by:**
`cockpitMode: "admin" | "customer"`

Stored in:
*   BrewLast.json
*   LocalStorage
*   JWT (future)
*   Runtime context

**3.2 Switch location**
Admin sees a toggle inside Command Bar → “Admin Mode / Customer Mode”
Customer never sees switch.

**5. Access Control Layer**
All Dev features now go through:
A) `ToolbeltGuard.ts`
Checks Tier permissions against active mode.
B) `IntentGatekeeper`
Determines if prompt is allowed in Customer Mode.
C) `SandboxProtection.ts`
Ensures sandbox never becomes visible in Customer Mode.
D) **API Protection**
API routes for:
*   `/api/brewassist`
*   `/api/brewtruth`
*   `/api/files/*`
*   `/api/mcp/*`
must check:
`if (mode === "customer" && request.isDestructive) { return 403; }`

**6. Integration With BrewTruth & Intent Gatekeeper**
**6.1 BrewTruth Behavior**
| Mode     | Truth Behavior                  |
| :------- | :------------------------------ |
| Admin    | Truth runs, visible, enforces risk gates |
| Customer | Truth disabled, no chip shown   |

**6.2 Intent Gatekeeper**
| Intent      | Admin Action      | Customer Action             |
| :---------- | :---------------- | :-------------------------- |
| ENGINEERING | Execute           | Block + “not supported in this mode” |
| GENERAL     | BrewChat          | BrewChat                    |
| KNOWLEDGE   | BrewCore          | BrewCore                    |
| RISK        | Run BrewTruth     | Block                       |
| OVERRIDE    | Allowed           | ✖ No access                 |

Customer mode cannot elevate privilege.

**7. Component-Level Implementation**
Below is EXACTLY what must change.

**7.1 `components/BrewCockpitCenter.tsx`**
Add:
*   Mode guard wrapper around Sandbox
*   Mode guard around ActionMenu
*   TruthChip conditional visibility
*   Remove Dev logs for customers

**7.2 `components/WorkspaceSidebarLeft.tsx`**
Wrap each button:
`if (mode === "customer") return null;`

Buttons that disappear:
*   Sandbox
*   Agents
*   File Ops
*   Repo Tools
*   Truth Test
*   Wizard triggers

**7.3 `components/ActionMenu.tsx`**
Replace menu with empty fragment if `mode=customer`.

**7.4 `lib/brewassist-engine.ts`**
Before running any Tier 2 or Tier 3 operations:
`if (mode === "customer") throw new Error("Protected operation");`

Before writing files, calling shell, or editing anything:
`ToolbeltGuard.verifyTier(2 or 3, mode);`

**7.5 `lib/toolbeltGuard.ts`**
Extend with:
`if (mode === "customer") { if (tier > 1) throw new Error("Not permitted in customer mode"); }`
