## S4.10a — MCP Tools Full Overhaul Blueprint

### Objective

To create a comprehensive blueprint for the full overhaul of the MCP Tools panel, detailing the architecture, user flows, and technical specifications for each wizard (File Operations, Git Command Center, Database Assistant, Research & Validation) and other recommended tools. This blueprint will serve as the foundational document for the implementation of S4.10.

### Deliverables

1.  **Detailed User Flows:**
    - Map out step-by-step user interactions for each MCP Tool wizard, including modal states, input fields, and confirmation steps.
    - Illustrate how context (highlighted code, selected files, project structure) influences tool behavior and suggestions.

2.  **Technical Specifications for Each Wizard:**
    - **File Operations Wizard:**
      - Create File: Directory tree selector, file type selection, optional templates (React Component, Next.js API Route, etc.), confirmation.
      - Delete File: Directory tree, confirmation with backup option.
      - Rename / Move File: Visual drag-and-drop, auto-undo.
      - Duplicate File, Create Folder, Upload Asset.
    - **Git Command Center Wizard:**
      - Provider selection (GitHub, GitLab, Bitbucket, local), authentication.
      - Guided prompts for Add, Commit, Push, Tag, Open PR.
      - Integration with BrewAssist to help complete PR templates.
    - **Database Assistant Wizard:**
      - Provider selection (Supabase, PostgreSQL, MySQL, MongoDB), credential input.
      - Schema validation, migration script generation, schema diffs, constraint suggestions, query optimization.
    - **Research & Validation Wizard:**
      - Triggered by highlighted text.
      - NIMs web research integration.
      - BrewTruth comparison score, model recommendations, confidence tier, patch suggestions.

3.  **Integration with BrewTruth and NIMs:**
    - Specify how BrewTruth scores, routing info, and provider cross-check indicators will be displayed within each wizard.
    - Detail how NIMs research will be initiated and its results processed and presented.

4.  **Error Handling and UI Feedback:**
    - Define consistent error handling mechanisms for all tool operations.
    - Specify meaningful UI feedback (e.g., loading states, success messages, error alerts) and fallback behaviors.

5.  **Future-Proof MCP Tools List:**
    - Refine the recommended list of MCP Tools, categorizing them (File Operations, Code Operations, Research & Validation, Repo Tools, Database Tools, Workflow Tools) and detailing their planned functionality.

### Next Steps

- Upon approval of this blueprint, proceed with the implementation of S4.10b.

## S4.10b — MCP Tools UI Implementation (Phase 1)

### Objective

To implement the user interface and core functionality for the first phase of the MCP Tools Overhaul, based on the approved S4.10a blueprint. This phase will focus on bringing the wizard modals to life with interactive elements and basic logic.

### Deliverables

1.  **Implement Core UI for Wizards:**
    - **`McpWizardModal.tsx`:** Enhance the generic modal component with improved styling and responsiveness.
    - **`FileWizard.tsx`:** Implement the UI for "Create File" (directory selector, file type, templates) and "Delete File" (directory tree, confirmation).
    - **`GitWizard.tsx`:** Implement the UI for "Add + Commit + Push Wizard" (repo provider selection, commit message input, branch selection).
    - **`DatabaseWizard.tsx`:** Implement the UI for "Database Connect" (provider selection, credential input).
    - **`ResearchWizard.tsx`:** Implement the UI for displaying NIMs research results and BrewTruth comparison.

2.  **Basic Interaction Logic:**
    - Wire up UI elements within each wizard to capture user input.
    - Implement placeholder functions for actions (e.g., `handleCreateFile`, `handleCommitAndPush`) that will eventually call backend APIs.

3.  **Context-Aware UI Elements:**
    - Implement logic to enable/disable MCP Tools buttons based on available context (e.g., highlighted text, active file).

4.  **BrewTruth Display Integration:**
    - Integrate placeholder UI elements within each wizard to display BrewTruth scores, routing info, and provider cross-check indicators.

### Next Steps

- Upon completion and verification of Phase 1 UI, proceed with backend API integration and advanced functionality in subsequent S4.10 phases.
