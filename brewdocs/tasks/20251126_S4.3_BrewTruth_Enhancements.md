# S4.3 – BrewTruth Engine Enhancements

**Project:** BrewExec / BrewAssist DevOps Cockpit
**Phase:** S4.3 – BrewTruth Engine Enhancements
**Date:** 2025‑11‑26
**Owner:** RB + Production ChatG

---

## 1. Goal

The primary goal of S4.3 is to further enhance the BrewTruth Engine, building upon the successful integration with BrewLast in S4.2. This phase will focus on improving the quality and utility of BrewTruth's evaluations, potentially through more sophisticated context injection, refined prompting strategies, and initial considerations for UI integration.

---

## 2. Proposed Enhancements

### 2.1 Contextual Awareness

*   **Inject broader context:** Explore ways to provide BrewTruth with more context beyond just the tool run details. This could include:
    *   Relevant code snippets from the project.
    *   Architectural documentation.
    *   Recent chat history with the user.
*   **Dynamic Prompting:** Develop more dynamic and adaptive prompting strategies for BrewTruth based on the type of tool run or the perceived risk level.

### 2.2 Refined Evaluation Metrics

*   **Granular Risk Levels:** Investigate if the `riskLevel` can be more granular or if additional dimensions of risk (e.g., security risk, performance risk) can be introduced.
*   **Actionable Feedback:** Improve the `flags` and `summary` fields to provide more actionable and specific feedback to the user or to BrewAssist itself.

### 2.3 Initial UI Integration Considerations

*   **Displaying Truth Reviews:** Begin planning how `truthReview` blocks will be displayed in the BrewAssist UI (e.g., in the Workpane, alongside tool run history).
*   **User Interaction with Reviews:** Consider how users might interact with these reviews (e.g., acknowledging flags, requesting re-evaluation).
